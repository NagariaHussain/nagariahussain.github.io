Introduction

The main purpose of the website is to provide the user with Articles and Tutorials on the following topics:
  1. Technology and Programming
  2. Travel and Tourism
  3. Food
  4. News: Short Articles
I chose to implement it as 5 page website which has a home page and one page for each of the above topics. The goal was to create a responsive website that provides its user with information in an organized and beautiful manner.

3 ways in which my site is accessible

  1. The alt tags of all the image elements contain a description of the image to help users with disability to visualize the image.
  2. I have used proper semantic and element tags to make the different parts of the website more accessible. For instance, use of article and figure tags are used instead of plan div tags.
  3. Use of aria labels to make mobile navigation more accessible.
    "class="nav-button hamburger hamburger--elastic" aria-label="Menu" aria-controls="navigation" aria-expanded="false"

3 things I had to learn/find out by myself when creating my site, and describe how I did that

  1. Flexbox: When I was digging more into layouts using CSS, I came across Flexbox and how it can make creating responsive layouts very easy. As it was not covered in the module, I learned it using the below resources:
    • CSS Grids and Flexbox for Responsive Web Design (course by Frontendmasters): View the course here
    • A complete guide to Flexbox (CSS-TRICKS): Here
  2. CSS Custom Variables: When I wanted to theme my website, i.e., set a different theme color for each page using JavaScript, the requirement was to set a primary color and change it based on the URL the user is currently on. I came across the concept of CSS Custom Properties (a.k.a. CSS Variables). I learned it using this article on MDN. I have used 3 variables to control the color for a page:
    1. --primary-color
    2. --primary-backdrop-color
    3. --secondary-color
    Now, if I want to change the primary color of a page, I can change it in just one place and every other occurence of the color changes automatically. I have used JavaScript to change the value associated with the variables based on the URL (relative path to be precise). For example, see the code snippet below:

    if (location.pathname === "/pages/shorts.html") {
    root.style.setProperty('--primary-color', "#ffc93c");
    root.style.setProperty('--primary-backdrop-color', "rgba(255, 201, 60, 0.6)");
    }

  3. Modal: Modals are part of many websites and are a way to display information only when it is required. So, I searched how to create a modal on Google and found many articles that showed how to do it. I learned to create a modal using this W3Schools article. I started with the code given on the above website and changed some CSS to match it with my web page. I also added show and hide animations to improve the UX.
  4. Carousel Slider: CSS Tricks tutorial and Flickity
  5. Date Object: The JavaScript Date object gives us access to many functions and data related to date and time. I used the MDN reference to learn about the different methods available on the Date object. I have used the Date object to get the current year and some DOM manipulation to set the copyright year to the current year. This way, we don't have to manually update the year next to the © icon every year:

    // Get current year
    const currentYear = new Date().getFullYear();
    // Update span text
    copyrightYearSpan.innerText = currentYear;
  6. Intersection Observer API: I have seen this cool effect on many websites, where, some elements animate into the view as we scroll the page. I wanted to implement this in my website. I looked for possible methods to do the same. Although, there are many external JS Libraries available for this task but using the IntersectionObserver API is native and hence I decided to implement the effect without the use of any external library. This effect can be seen on the home page: the category divs and avatars animate as you scroll to their position.
